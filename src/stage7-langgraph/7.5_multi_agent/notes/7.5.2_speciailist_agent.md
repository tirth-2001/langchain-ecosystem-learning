# **7.5.2 — Specialist Agents (Researcher + Writer)**

We will use the same 5-part structure.

This is one of the most powerful patterns in multi-agent systems, because it mirrors how real teams work:

```
User → Researcher Agent → Writer Agent → Final Answer
```

Let’s begin.

---

# ⭐ 1) GOAL

Build a two-step **specialist agent workflow**:

- **Researcher Agent**
  → Finds & synthesizes knowledge from context (or RAG, tools, or even web API later)

- **Writer Agent**
  → Converts the research output into a clean final presentation (blog, summary, email, report)

This is the template for:

- Blog article generators
- Q&A engines
- Multi-stage pipelines
- Multi-agent “roles” (CrewAI-style)

---

# ⭐ 2) THEORY & CONCEPTS

### 🧠 Why multiple specialists?

LLMs perform best when given **explicit roles**, because roles shape behavior.

Examples:

- “Researcher” gives factual, structured notes
- “Writer” creates a polished narrative
- “Planner" breaks down tasks
- “Critic" checks quality

This reduces hallucinations and improves output quality.

---

### 🔄 Flow Structure

```
User Input
   ↓
Researcher Agent
   ↓
Writer Agent
   ↓
Final Output
```

State fields:

```ts
{
  input: string
  research: string
  article: string
}
```

---

### 🧩 Researcher Agent Responsibilities

- Extract key ideas
- Expand relevant facts
- Produce structured reference notes

It acts like a **knowledge synthesizer**.

---

### 📝 Writer Agent Responsibilities

- Convert research → polished text
- Maintain tone, clarity, formatting
- Insert bullets, sections, summaries

---

### 🔗 Extending Later

You can add:

- **SEO Agent**
- **Fact-check Agent**
- **Critic Agent**
- **Editor Agent**

This forms a _full publishing team_.

---

## **3.3 — Researcher Agent**

Goal → turn raw idea into structured notes.

```ts
const researchPrompt = ChatPromptTemplate.fromTemplate(`
You are a RESEARCHER agent.

Your job:
- Analyze the user's request
- Produce structured research notes
- Include facts, bullet points, details
- No storytelling

User request:
"{input}"

Output ONLY structured research notes.
`)

const researchChain = RunnableSequence.from([researchPrompt, llm, (out) => out.content.trim()])

async function researcherNode(state: typeof StateAnnotation.State) {
  console.log('🔍 Researcher working on:', state.input)
  const research = await researchChain.invoke({ input: state.input })
  return { research }
}
```

---

## **3.4 — Writer Agent**

Goal → turn research into polished article.

```ts
const writerPrompt = ChatPromptTemplate.fromTemplate(`
You are a WRITER agent.

Using the research notes:
{research}

Write a well-structured article.

Guidelines:
- Use headings
- Bullet points
- Clear flow
- No hallucinations
- Use only the provided research

Return ONLY the article.
`)

const writerChain = RunnableSequence.from([writerPrompt, llm, (out) => out.content.trim()])

async function writerNode(state: typeof StateAnnotation.State) {
  console.log('✍️ Writer creating article...')
  const article = await writerChain.invoke({ research: state.research })
  return { article }
}
```

---

## **3.5 — Graph Wiring**

```ts
const workflow = new StateGraph(StateAnnotation)
  .addNode('researcher', researcherNode)
  .addNode('writer', writerNode)
  .addEdge(START, 'researcher')
  .addEdge('researcher', 'writer')
  .addEdge('writer', END)

const app = workflow.compile()
```

# ⭐ 4) FINAL PLUG-N-PLAY FILE

### 📌 `specialist-agents.ts`

```ts
import { Annotation, StateGraph, START, END } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { ChatPromptTemplate } from '@langchain/core/prompts'
import { RunnableSequence } from '@langchain/core/runnables'
import 'dotenv/config'

/**
 * 1. State
 */
const StateAnnotation = Annotation.Root({
  input: Annotation<string>(),
  research: Annotation<string | null>(),
  article: Annotation<string | null>(),
})

/**
 * 2. LLM
 */
const llm = new ChatOpenAI({
  modelName: 'gpt-4o-mini',
  temperature: 0.4,
})

/**
 * 3. Researcher Agent
 */
const researchPrompt = ChatPromptTemplate.fromTemplate(`
You are a RESEARCHER agent.

Analyze the user's request:
"{input}"

Produce structured research notes:
- Key facts
- Bullet points
- Important details
- No storytelling

Return ONLY research notes.
`)

const researchChain = RunnableSequence.from([researchPrompt, llm, (out) => out.content.trim()])

async function researcherNode(state: typeof StateAnnotation.State) {
  console.log('🔍 Researcher analyzing...')
  const research = await researchChain.invoke({ input: state.input })
  return { research }
}

/**
 * 4. Writer Agent
 */
const writerPrompt = ChatPromptTemplate.fromTemplate(`
You are a WRITER agent.

Using this research:
{research}

Produce a well-written article with:
- Headings
- Sections
- Bullet points
- No hallucinations
- Based ONLY on research provided

Return ONLY the article.
`)

const writerChain = RunnableSequence.from([writerPrompt, llm, (out) => out.content.trim()])

async function writerNode(state: typeof StateAnnotation.State) {
  console.log('✍️ Writer composing...')
  const article = await writerChain.invoke({ research: state.research })
  return { article }
}

/**
 * 5. Graph
 */
const workflow = new StateGraph(StateAnnotation)
  .addNode('researcher', researcherNode)
  .addNode('writer', writerNode)
  .addEdge(START, 'researcher')
  .addEdge('researcher', 'writer')
  .addEdge('writer', END)

const app = workflow.compile()

/**
 * 6. Demo
 */
async function main() {
  const result = await app.invoke({
    input: 'Write a blog about how LangGraph improves agent workflow reliability.',
  })

  console.log('\n📝 FINAL ARTICLE:\n')
  console.log(result.article)
}

main().catch(console.error)
```

# ⭐ 5) NOTES & TIPS

### ✔ “Roles” increase accuracy

LLMs behave more predictably when given explicit job functions.

### ✔ Specialists reduce hallucinations

Researcher sticks to facts → Writer sticks to narrative explains why CrewAI works well.

### ✔ Debug easily

Use event streaming to inspect both agents' outputs.

### ✔ Expand later

Add more roles:

- Editor Agent
- Critic Agent
- SEO Agent
- Detail Expander
- Query Rewriter
- Fact Checker

### ✔ Multi-agent patterns scale well

This is the base of multi-agent orchestration.
