# **7.5.3 — Shared Memory Across Agents**

---

# ⭐ 1) GOAL — What You’ll Build

Create **multiple collaborating agents** (Researcher, Writer, Critic, etc.)
that **share a global memory state**, so each agent can see the entire conversation history and build on top of it.

You’ll learn how to:

- Store shared long-term knowledge inside LangGraph state
- Allow multiple nodes (agents) to **read/write** the same memory
- Build cross-agent brain:
  _Research notes → Writer response → Critic feedback → Improved output_

The final architecture looks like:

```
User Input
     ↓
Researcher Agent       (writes to shared memory)
     ↓
Writer Agent           (reads memory, writes improved text)
     ↓
Critic Agent           (reads + adds feedback)
     ↓
Writer Agent (again)   (optional improvement loop)
```

---

# ⭐ 2) THEORY & CONCEPTS

### 🔥 Why Shared Memory?

Because **agents need context about each other’s work**.

Examples:

- Researcher → produces notes
- Writer → must read those notes
- Critic → must see both notes AND writer’s draft
- Editor → must see everything

Relying only on LLM prompts is not reliable.
Using **shared state fields** gives deterministic behavior.

---

### 💡 LangGraph State = Global Memory

Each agent node updates part of the state:

```
input           ← user query
research        ← written by Researcher
article         ← written by Writer
feedback        ← written by Critic
improvedArticle ← written by Writer (2nd pass)
```

LangGraph merges state updates using its reducers.
This creates a **global shared notebook** for all agents.

---

### 🔄 Shared Memory Pattern

We will maintain a single state:

```ts
{
  input: string
  research: string[]
  drafts: string[]
  feedback: string[]
}
```

Each agent simply _appends_ something to that list.

---

### 🧠 Benefits

- Multi-agent reasoning becomes stable
- Avoids hallucination between agents
- Creates persistent knowledge across the flow
- Enables loops & collaboration

# ⭐ 3) CODE SNIPPETS (Step-by-Step)

---

## **3.1 — Define Shared Memory State**

We’ll use array reducers to append output:

```ts
const StateAnnotation = Annotation.Root({
  input: Annotation<string>(),
  research: Annotation<string[]>({
    reducer: (curr, update) => [...(curr ?? []), ...(update ?? [])],
  }),
  drafts: Annotation<string[]>({
    reducer: (curr, update) => [...(curr ?? []), ...(update ?? [])],
  }),
  feedback: Annotation<string[]>({
    reducer: (curr, update) => [...(curr ?? []), ...(update ?? [])],
  }),
})
```

---

## **3.2 — Researcher Agent (writes to shared memory)**

```ts
const researchPrompt = ChatPromptTemplate.fromTemplate(`
You are a RESEARCHER agent.
Provide structured notes for: "{input}"

Output short factual bullet points only.
`)

const researchChain = RunnableSequence.from([researchPrompt, llm, (out) => out.content.trim()])

async function researcherNode(state: typeof StateAnnotation.State) {
  const note = await researchChain.invoke({ input: state.input })
  return { research: [note] } // append via reducer
}
```

---

## **3.3 — Writer Agent (reads memory)**

```ts
const writerPrompt = ChatPromptTemplate.fromTemplate(`
You are a WRITER agent.
Using the research notes:
{research}

Write one short paragraph.
`)

const writerChain = RunnableSequence.from([writerPrompt, llm, (out) => out.content.trim()])

async function writerNode(state: typeof StateAnnotation.State) {
  const text = await writerChain.invoke({
    research: state.research.join('\n'),
  })
  return { drafts: [text] } // append
}
```

---

## **3.4 — Critic Agent (writes feedback)**

```ts
const criticPrompt = ChatPromptTemplate.fromTemplate(`
You are a CRITIC.
Give constructive feedback for this draft:

{draft}

Keep it brief.
`)

const criticChain = RunnableSequence.from([criticPrompt, llm, (out) => out.content.trim()])

async function criticNode(state: typeof StateAnnotation.State) {
  const lastDraft = state.drafts[state.drafts.length - 1]
  const fb = await criticChain.invoke({ draft: lastDraft })
  return { feedback: [fb] }
}
```

# ⭐ 4) FINAL PLUG-N-PLAY CODE — **multi-agent-shared-memory.ts**

```ts
import { Annotation, StateGraph, START, END } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { ChatPromptTemplate } from '@langchain/core/prompts'
import { RunnableSequence } from '@langchain/core/runnables'
import 'dotenv/config'

/**
 * 1. Shared State
 */
const StateAnnotation = Annotation.Root({
  input: Annotation<string>(),

  research: Annotation<string[]>({
    reducer: (curr, update) => [...(curr ?? []), ...(update ?? [])],
  }),

  drafts: Annotation<string[]>({
    reducer: (curr, update) => [...(curr ?? []), ...(update ?? [])],
  }),

  feedback: Annotation<string[]>({
    reducer: (curr, update) => [...(curr ?? []), ...(update ?? [])],
  }),
})

/**
 * 2. Base Model
 */
const llm = new ChatOpenAI({
  modelName: 'gpt-4o-mini',
  temperature: 0.4,
})

/**
 * 3. Researcher Node
 */
const researchPrompt = ChatPromptTemplate.fromTemplate(`
You are a RESEARCHER agent.
Provide structured notes for: "{input}"

Return ONLY bullet points.
`)

const researchChain = RunnableSequence.from([researchPrompt, llm, (o) => o.content.trim()])

async function researcherNode(state: typeof StateAnnotation.State) {
  const note = await researchChain.invoke({ input: state.input })
  return { research: [note] }
}

/**
 * 4. Writer Node
 */
const writerPrompt = ChatPromptTemplate.fromTemplate(`
You are a WRITER agent.

Research notes:
{research}

Write 1 short paragraph using ONLY these notes.
`)

const writerChain = RunnableSequence.from([writerPrompt, llm, (o) => o.content.trim()])

async function writerNode(state: typeof StateAnnotation.State) {
  const notes = state.research.join('\n')
  const text = await writerChain.invoke({ research: notes })
  return { drafts: [text] }
}

/**
 * 5. Critic Node
 */
const criticPrompt = ChatPromptTemplate.fromTemplate(`
You are a CRITIC.
Give brief constructive feedback for this draft:

{draft}
`)

const criticChain = RunnableSequence.from([criticPrompt, llm, (o) => o.content.trim()])

async function criticNode(state: typeof StateAnnotation.State) {
  const last = state.drafts[state.drafts.length - 1]
  const fb = await criticChain.invoke({ draft: last })
  return { feedback: [fb] }
}

/**
 * 6. Build Graph
 */
const workflow = new StateGraph(StateAnnotation)
  .addNode('researcher', researcherNode)
  .addNode('writer', writerNode)
  .addNode('critic', criticNode)
  .addEdge(START, 'researcher')
  .addEdge('researcher', 'writer')
  .addEdge('writer', 'critic')
  .addEdge('critic', END)

const app = workflow.compile()

/**
 * 7. Demo
 */
async function main() {
  const result = await app.invoke({
    input: 'Explain how LangGraph improves agent reliability.',
  })

  console.log('\n🔍 RESEARCH NOTES:')
  console.log(result.research.join('\n\n'))

  console.log('\n✍️ DRAFT:')
  console.log(result.drafts.join('\n\n'))

  console.log('\n🧪 FEEDBACK:')
  console.log(result.feedback.join('\n\n'))
}

main().catch(console.error)
```

# ⭐ 5) NOTES & TIPS

### ✔ Arrays = Best Shared Memory Format

Because multiple agents can append without overwriting each other.

### ✔ Add summaries later

Add a `summary` field compressed by a summarizer node.

### ✔ All agents can read/write

This creates true collaboration.

### ✔ You can add loops

Writer → Critic → Writer → Critic, until “quality OK”.

### ✔ This architecture = CrewAI but deterministic

CrewAI = "LLM-freestyle conversation"
LangGraph = "deterministic, structured agent collaboration".

### ✔ Later you can store memory in Mongo/Redis

Perfect for long-running workflows.
