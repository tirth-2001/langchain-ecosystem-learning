# ✅ **7.5.1 — Two-Agent System (Planner + Executor)**

We will follow the same 5-part structure.

---

# ⭐ **1) GOAL**

Build a minimal **two-agent crew** inside LangGraph:

```
User → Planner Agent → Executor Agent → Output
```

Where:

- **Planner Agent**
  Breaks the user request into a clear actionable step.

- **Executor Agent**
  Performs the action (LLM-based execution, tool usage, or transformation).

This is the smallest but most fundamental multi-agent architecture.
Almost every system like **CrewAI, Devin, OpenAI Orchestrator, AutoGPT** starts with this pattern.

---

# ⭐ **2) THEORY & CONCEPTS**

## 🧠 Why Two Agents?

Because:

- One agent _decides what to do_ (Planner).
- Another _executes the plan_ (Executor).

This separation reduces hallucinations and yields more deterministic behavior.

---

## 💡 LangGraph Model

We use:

- **Nodes as agents**.
- **Edges to route tasks**.
- **State to store:**

  - user input
  - plan
  - execution output

State shape:

```ts
{
  input: string
  plan: string
  output: string
}
```

This is effectively:

```
User → “What should we do?” → Planner
Planner → “Do this step” → Executor
Executor → “Here is the final response”
```

---

## 🏗️ Execution Flow

1. **User query** (ex: _“Summarize this text and extract keywords”_)
2. **Planner**:
   → _“Step 1: Summarize. Step 2: Extract keywords.”_
3. **Executor**:
   → Performs the instructions and outputs the final answer.
4. **Graph ends**

---

## ✨ Why This Matters

This architecture enables:

- Task decomposition
- Multi-step workflows
- Multi-agent collaboration
- Supervisor patterns
- Long-running autonomy

This is the foundation for:

✔ Agent swarms
✔ Hierarchical workflows
✔ Role-based teams
✔ Tool routing via Planner
✔ Self-correction loops

## **3.3 — Planner Node**

This agent creates a structured plan.

```ts
const plannerPrompt = ChatPromptTemplate.fromTemplate(`
You are a planning agent.
Generate a SINGLE clear step to solve: "{task}"

Examples:
- "Summarize the text."
- "Fetch weather then rewrite."

Output ONLY the next best step.
`)

const plannerChain = RunnableSequence.from([plannerPrompt, llm, (out) => out.content.trim()])

async function plannerNode(state: typeof StateAnnotation.State) {
  console.log('🧠 Planner received task:', state.input)

  const plan = await plannerChain.invoke({ task: state.input })
  console.log('📋 Planner generated:', plan)

  return { plan }
}
```

## **3.4 — Executor Node**

This agent performs the step.

```ts
const executorPrompt = ChatPromptTemplate.fromTemplate(`

"{plan}"

- Do NOT explain what you are doing.
- Output ONLY the final result.

const executorChain = RunnableSequence.from([executorPrompt, llm, (out) => out.content.trim()])
```

# ⭐ **4) FINAL PLUG-N-PLAY FILE (FULL CODE)**

Copy-paste → Run → Works.

---

### 📌 `two-agent-crew.ts`

```ts
import { Annotation, StateGraph, END, START } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { ChatPromptTemplate } from '@langchain/core/prompts'
import { RunnableSequence } from '@langchain/core/runnables'
import 'dotenv/config'

/**
 * 1. State definition
 */
const StateAnnotation = Annotation.Root({
  input: Annotation<string>(),
  plan: Annotation<string | null>(),
  output: Annotation<string | null>(),
})

/**
 * 2. LLM setup
 */
const llm = new ChatOpenAI({
  modelName: 'gpt-4o-mini',
  temperature: 0.4,
})

/**
 * 3. Planner Agent
 */
const plannerPrompt = ChatPromptTemplate.fromTemplate(`
You are a planning agent.
Generate ONE clear next step for: "{task}"
Be short and actionable.
`)

const plannerChain = RunnableSequence.from([plannerPrompt, llm, (out) => out.content.trim()])

async function plannerNode(state: typeof StateAnnotation.State) {
  console.log('🧠 Planner received:', state.input)
  const plan = await plannerChain.invoke({ task: state.input })
  console.log('📋 Generated plan:', plan)
  return { plan }
}

/**
 * 4. Executor Agent
 */
const executorPrompt = ChatPromptTemplate.fromTemplate(`
Execute this instruction:
"{plan}"

Return ONLY the result.
`)

const executorChain = RunnableSequence.from([executorPrompt, llm, (out) => out.content.trim()])

async function executorNode(state: typeof StateAnnotation.State) {
  console.log('⚙️ Executor executing:', state.plan)
  const output = await executorChain.invoke({ plan: state.plan })
  console.log('🟢 Executor result:', output)
  return { output }
}

/**
 * 5. Build Graph
 */
const workflow = new StateGraph(StateAnnotation)
  .addNode('planner', plannerNode)
  .addNode('executor', executorNode)
  .addEdge(START, 'planner')
  .addEdge('planner', 'executor')
  .addEdge('executor', END)

const app = workflow.compile()

/**
 * 6. Demo
 */
async function main() {
  const result = await app.invoke({
    input: 'Summarize this text and extract keywords: LangGraph helps build agent workflows.',
  })

  console.log('\n🚀 Final Output:', result.output)
}

main().catch(console.error)
```

# ⭐ **5) NOTES & TIPS**

### ✔ Planner simplifies reasoning

LLMs perform better when planning and execution are separated.

### ✔ Multistep loops

You can extend this into a loop:

```
Planner → Executor → Checker → Planner → ...
```

### ✔ Replace executor with:

- A tool node
- A RAG node
- Another agent

### ✔ Extend state with progress

Add fields like `stepsCompleted`, `history`, etc.

### ✔ This is now a real foundation for:

- CrewAI-like multi-agent systems
- Devin-like autonomous agents
- Workflow automations
