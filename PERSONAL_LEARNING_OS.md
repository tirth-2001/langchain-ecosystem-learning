## 3️⃣ The 5-Step Learning Loop (Your Core Engine)

### 🔁 Step 1 — Intent Lock

Before starting anything, answer **one sentence**:

> “I am learning X in order to understand Y, not to master Z.”

Example:

> “I am learning LangGraph to understand deterministic agent orchestration, not to memorize APIs.”

If you can’t say this → **don’t start**.

---

### 🧠 Step 2 — Mental Model First

Before code, explicitly define:

- What is the **state**?
- What changes it?
- Who controls flow?
- Where can it fail?

Write it as **1–3 bullet points**, not paragraphs.

---

### 🧩 Step 3 — Minimal Runnable Core

You only write code that:

- Demonstrates the mental model
- Is runnable in isolation
- Can break in visible ways

❌ No “full apps”
❌ No premature abstractions

---

### 🔍 Step 4 — Edge Case Injection

Intentionally ask:

- What if input is missing?
- What if state is partial?
- What if it loops?
- What if it resumes?

This is where **your learning differentiates**.

---

### 🧊 Step 5 — Abstraction Freeze

Once it works and makes sense:

> “I will not think about this again unless building a product.”

Document:

- 3 invariants
- 2 failure modes
- 1 transfer insight

Then **move on**.

## 8️⃣ Framework Evaluation Playbook (Future-Proofing)

When you encounter a new framework (CrewAI, AutoGen, Devin, etc.):

Ask only 6 questions:

1. What is the **state model**?
2. Who controls **flow**?
3. Where is **memory stored**?
4. How is **failure handled**?
5. Is execution **deterministic**?
6. What is hidden from me?

If you can answer these → you’ve mastered it conceptually.
